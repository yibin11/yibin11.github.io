知识点：数据驱动视图

M-V-VM：model的数据变，view也跟着变；只要view变化，model也跟着变化

在控制台修改变量的值，页面会立马跟着变化 —》以后只需要修改变量值即可，不需要操作dom了

  1. 关于“修改变量值，页面立马变化，不需要操作DOM”
    • 完全正确！ 这是 Vue 的核心魅力。Vue 帮你在底层监听了数据的变化，一旦数据变了，Vue 会自动去更新对应的 DOM。开发者只需要专注于“数据是什么”，而不需要像 jQuery 时代那样去写 document.getElementById().innerHTML = ...。
    • 你描述的这个特性，在 Vue 中准确的叫法是 响应式（Reactivity） 或 单向数据流（数据变化导致页面变化）。
  2. 关于 vm._data 和 vm.变量名
    • Vue2 实例化后，会把 data 里的属性挂载到 vm._data 下。为了方便开发者，Vue 底层使用 Object.defineProperty 做了代理（Proxying），让你直接访问 vm.s 就能拿到 vm._data.s 的值。
  3. 关于“后台或数据库数据变化，前端就变？不是”
    • Vue 只是一个前端框架，它运行在用户的浏览器内存里。它只能监听到浏览器里 JavaScript 变量的变化。
    • 数据库变了，前端不会自动知道。如果想要实现“数据库变了，前端页面立马跟着变”，需要借助网络协议主动通知前端，比如使用 WebSocket 或者 SSE (Server-Sent Events) 技术。

这些知识点在 Vue3 中怎么表示？

Vue3 相比 Vue2，最大的变化是底层响应式原理的改变和API 写法的改变。

1. Vue3 推荐的新写法：

组合式 API（Composition API）

这是 Vue3 的精髓。在 Vue3 中，我们不再依赖 this 或 vm 实例的魔力，而是引入了独立的响应式 API（ref 和 reactive）。

注意：如果是上面这种 setup 写法，在控制台修改变量时，由于 ref 会把值包装在一个对象里，你在控制台可能需要输入 vm.b1 = false 来改变它（app.mount 返回的根组件实例依然做了代理）。在 JS 代码内部，你需要写 b1.value = false。

2. 底层原理的巨大改变

  • Vue2 的 vm._data： 底层是用 Object.defineProperty 劫持了对象的 getter 和 setter 来实现页面更新的。它的缺点是无法监听到给对象新增属性（比如本来没有 c，你突然 vm.c = 1，页面是不会变的）。
  • Vue3 的响应式： 放弃了 Object.defineProperty，改用了 ES6 的 Proxy 对象。Proxy 会包裹整个数据对象。
    • 在 Vue3 中，没有 vm._data 了。如果你打印 vm.$data（Vue3 获取原始数据的方式），你会发现它是一个 Proxy 对象。
    • Proxy 的好处是：不管你新增属性、删除属性、还是操作数组的索引，它都能完美拦截到，从而触发页面更新。

总结

只需要记住：

  1. 概念没变： 依然是数据驱动视图，改变量页面就变，不需要操作 DOM。
  2. 入口变了： new Vue() 变成了 Vue.createApp().mount()。
  3. data 变了： Vue3 的 data 必须写成函数 data() { return {} }。
  4. 进阶方向： 以后学 Vue3，重点去学 Composition API（组合式 API） 中的 ref 和 reactive，这才是 Vue3 真正的核心玩法。

2.事件指令

在 Vue3 中，关于模板标签里的指令（v-on、@click、v-if、v-show），它们的用法 100% 没有任何变化！ 你的这部分笔记可以直接无缝带入 Vue3。

在 Vue3 中，真正发生变化的是 <script> 里面的代码写法。也就是我们如何定义变量和函数，以及如何修改变量。

组合式 API 写法
在这个写法里，我们要抛弃 data、抛弃 methods、甚至抛弃 this！ 所有的变量和函数都写在一个叫 setup 的地方。

总结：知识点进阶清单

当你从 Vue2 进阶到 Vue3 的“组合式写法”时，你需要刷新一下思维：

  1. 指令没变：@click、v-if 继续大胆用。
  2. 结构变了：不再区分 data 区域和 methods 区域，全部写在一起（就像写普通的 JavaScript 代码一样）。
  3. 变量变了：为了让数据改变时页面能自动变，必须用 ref() 包裹初始值。
  4. this 消失了：Vue2 中到处都是 this.xxx。Vue3 的 setup 中没有 this，你想改 zs，直接修改 zs.value 即可。

（小提示：在 <script> 里修改变量必须写 zs.value，但是 Vue 很贴心，在 HTML 模板里写 v-if="zs" 即可，不需要写 v-if="zs.value"，Vue 在模板里会自动帮你脱掉 value 的外壳）。
